# Twitter/X Platform Rules

## Character Limits

- Free accounts: 280 characters per tweet
- Premium accounts: up to 25,000 characters
- Always target free-tier limits unless the user specifies otherwise

## Optimal Length

- Sweet spot: 71-100 characters for maximum engagement
- Tweets under 100 characters get 17% more engagement
- Default to short and punchy. Only go longer when context demands it.

## Post Structure (Default Format)

```
[Hook — bold claim, question, or surprising stat] [emoji]

[What's new / what's happening — 1-2 sentences explaining the value] [emoji]

[CTA — single clear action] [emoji]
```

Each of the three sections carries 1-2 emojis. This structure catches attention during scroll.

## Thread Rules

Threads get 3x more engagement than single long tweets. Offer a thread alternative when the content is too rich for a single tweet.

Thread structure:
- Tweet 1: Hook — must stand alone and compel "Show thread" clicks
- Each tweet: one clear point, concise
- Number tweets explicitly (1/N, 2/N, etc.)
- Final tweet: CTA + link
- Keep each tweet under 280 chars

## Hooks That Work

- Specific, surprising statistic
- Provocative question
- Bold contrarian statement
- Clear value proposition ("We just shipped X. Here's why it matters.")
- "Build in public" transparency ("We spent 3 months on this. Here's what we learned.")

## Tone

- Conversational, direct, like DM-ing a friend
- Avoid corporate jargon ("synergies", "cutting-edge", "excited to announce")
- Founder-voice and authenticity perform best for dev tools
- Problem-first framing > generic "We're live!" announcements

## Hashtags

Do NOT include hashtags on Twitter/X. They no longer meaningfully drive discovery on this platform.

## Emojis

1-2 per section (3 sections = up to 6 total). Use structural emojis that catch the eye during scroll — arrows, checkmarks, indicators. Avoid decorative emoji spam.

## CTAs

Single, soft CTA. Examples:
- "Try it now →"
- "Link in thread"
- "Star it on GitHub →"

## Posting Times

- Best: Tuesday-Thursday, 12-6 PM local time
- Tech/developer audience peaks during midday-afternoon pulse-check window
- Avoid weekends for B2B content
